#  Indian Poker: Tazza Edition (인디언 포커: 타짜 에디션)

---

## 1.  프로젝트 기획 및 설명

###  기획 의도
단순히 운에 의존하는 일반적인 인디언 포커 게임에 **영화 <타짜>의 요소를 결합**하여 더 긴장감 넘치는 게임을 만들고자 했습니다. 
내 패를 보지 못하는 인디언 포커의 기본 규칙에, '밑장빼기'와 '의심하기'라는 심리전 시스템을 도입하여 플레이어가 단순 확률 계산을 넘어 상대방과의 수 싸움을 즐길 수 있도록 기획했습니다.

###  창의적 요소 (차별점)
1.  **타짜 시스템 (Cheating Mechanics):**
    * 플레이어와 컴퓨터 모두 딜러가 되었을 때 **'기술(밑장빼기)'**을 시도할 수 있습니다.
    * 성공 시 원하는 좋은 패를 가져올 수 있지만, 기본 성공 확률은 30%로 설정하여 리스크를 부여했습니다.
2.  **상호 의심 시스템:**
    * 상대방이 패를 섞을 때 수상함을 느꼈다면 **"동작 그만!"**을 외쳐 기술 여부를 확인할 수 있습니다.
    * 성공 시 즉시 승리하지만, 생사람을 잡을 경우(실패 시) 페널티를 받게 되어 신중한 선택이 필요합니다.

## 2.  주요 기능& 룰 설명

| 기능 | 설명 |
| :--- | :--- |
| **카드 시스템** | ♠, ♦, ♥, ♣ 문양과 A~K 랭크를 가진 카드 객체 구현 (ASCII Art 활용) |
| **게임 진행** | 칩 베팅, 콜(Call), 레이즈(Raise), 폴드(Fold) 등 텍사스 홀덤의 표준 베팅 룰 적용 |
| **딜러 페이즈** | 매 라운드 딜러가 바뀌며, 정직하게 섞을지 기술을 쓸지 선택 가능 |
| **AI 알고리즘** | 컴퓨터는 자신의 패와 베팅 금액을 분석하여 확률적으로 의심, 베팅, 포기를 결정 |
| **승패 판정** | 칩이 0이 되면 파산(패배), 상대방을 파산시키면 승리 |

---

### 2-1. 기본 인디언 포커 룰
* **패의 크기:** A(가장 높음) > K > Q > J > 10 > ... > 2(가장 낮음) 순으로 숫자가 높은 쪽이 이깁니다.
* **카드 분배:** 플레이어와 컴퓨터가 각각 1장의 카드를 받습니다.
* **승부 방식:** **상대방의 카드는 공개**되어 볼 수 있지만, **자신의 카드는 보이지 않습니다.** 상대방이 가진 패를 추측하며 베팅하는 것이 핵심입니다.

### 2-2. 타짜 에디션 특수 룰
| 구분 | 설명 |
| :--- | :--- |
| **기술 (밑장빼기)** | 딜러(카드를 섞는 사람)는 30% 확률로 기술을 사용할 수 있습니다. 성공 시 자신에게 좋은 패, 상대에게 나쁜 패를 줄 수 있습니다. |
| **의심 (심리전)** | 상대방이 카드를 섞을 때 '밑장빼기냐?'라고 의심할 수 있습니다. <br> - **의심 성공:** 상대의 기술이 발각되어 즉시 승리합니다. <br> - **의심 실패:** 상대방이 정직했는데 의심했다면 페널티(5칩)를 잃습니다. |

### 2-3. 베팅 용어
| 용어 | 행동 |
| :--- | :--- |
| **Check (체크)** | 낼 돈이 없을 때 칩 소모 없이 턴을 넘깁니다. (상대방이 베팅하지 않았을 경우) |
| **Bet/Raise (베팅/레이즈)** | 판돈을 키우기 위해 칩을 추가로 겁니다. |
| **Call (콜)** | 상대방이 건 칩만큼 내고 승부(카드 오픈)를 봅니다. |
| **Fold (폴드)** | 패가 안 좋다고 판단하면 포기합니다. 현재까지 건 칩은 모두 잃게 됩니다. |

## 4.  개발 환경 및 기술 스택 (Tech Stack)

* **Language:** Python 3.10+
* **Libraries:** * `random`: 카드 셔플 및 AI 확률 계산
    * `time`: 게임 진행의 텐션 조절 (딜레이 효과)
    * `os`: 화면 클리어 기능
    * `unicodedata`: 한글/특수문자 출력 시 텍스트 정렬 보정

## 5. 후기
처음엔 간단한 카드 게임을 만들려고 했다가, '타짜'의 요소를 넣게 되면 더욱 흥미로울 것 같다는 생각을 했습니다. 그러다 보니까 난이도가 더 높아지고 시간도 많이 걸려서 힘들긴 했지만 꽤 재밌는 시간이었습니다. 
아쉬운 점은, 플레이어가 두 명이 한정이라는 점이라 나중에는 플레이어 수를 늘려서 해보고 싶다는 생각이 들었습니다.
